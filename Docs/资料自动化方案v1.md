## 一、项目概述

### 1.1 目标
通过CodeAgent技能体系，实现资料开发工作的标准化、自动化，降低理解成本，提升交付质量。

### 1.2 核心价值
- **零理解成本**：新开发者按规格输入即可生成标准交付件
- **质量保障**：自动化检查和测试，提前识别问题
- **知识沉淀**：FAQ知识库，减少重复支撑工作
- **一致性保障**：跨仓版本信息自动校验

|--------|------|------|----------|
| 应用开发指南 | MD文件 | 开发者接入指南 | CodeHub |
| 元服务开发指南 | MD文件 | 元服务接入指南 | CodeHub |

| systemAPI资料 | 内部文档 | 包含声明、概念、错误码、权限、UI体验等 | 内部平台 |

## 四、模块化Skill架构设计

### 4.1 设计原则

采用**模块化、原子化**的设计思路，将复杂能力拆分为独立的小skill，通过组合实现大skill功能。

### 4.2 架构优势

#### 4.2.1 单一职责
- 每个skill专注于一个具体任务
- 易于理解和维护

#### 4.2.2 可复用性
- 小skill可以被多个大skill调用
- 避免重复开发

#### 4.2.3 易于测试
- 小skill更容易验证正确性
- 问题定位更精准

#### 4.2.4 灵活组合
- 大skill可以像搭积木一样组合小skill
- 适应不同场景需求

### 4.3 架构层次

```
┌─────────────────────────────────────────────┐
│     综合Skill：开发指南生成器               │
│  （输入：d.ts + 注释 + 需求说明）            │
└──────────┬──────────────────────────────────┘
           │ 调用
           ├──────────────────────────────────┐
           │                                  │
┌──────────▼────────┐  ┌─────────────────────▼──────────────┐
│ 规范检查Skill     │  │ 内容生成Skill                    │
│ - 检查清晰性      │  │ - 生成章节内容                   │
│ - 检查完整性      │  │ - 生成示例代码                   │
│ - 检查正确性      │  │ - 生成流程图                     │
└───────────────────┘  └──────────────────────────────────┘
           │                                  │
           └────────────┬─────────────────────┘
                        │
           ┌────────────▼────────────┐
           │ 一致性检查Skill         │
           │ - 跨仓版本一致性        │
           │ - 交付件内容一致性      │
           └─────────────────────────┘
```

### 4.4 Skill分类

#### 4.4.1 基础检查类Skill（原子能力）
- `checking-docs-guide-quality`：开发指南规范检查
- `checking-sample-code-quality`：示例代码质量检查
- `checking-version-consistency`：版本一致性检查
- `checking-api-completeness`：API完整性检查

#### 4.4.2 内容生成类Skill（原子能力）
- `generating-doc-section`：生成文档章节
- `generating-sample-code`：生成示例代码
- `generating-flowchart`：生成流程图
- `generating-faq-entry`：生成FAQ条目

#### 4.4.3 综合处理类Skill（组合能力）
- `generating-dev-guide`：开发指南生成器
  - 调用：`checking-docs-guide-quality` + `generating-doc-section` + `checking-version-consistency`
- `generating-systemapi-doc`：systemAPI资料生成器
  - 调用：`checking-api-completeness` + `generating-doc-section` + `generating-sample-code`
- `maintaining-sample-code`：sampleCode维护助手
  - 调用：`checking-sample-code-quality` + 问题修复逻辑

### 4.5 工作模式

#### 4.5.1 独立使用模式
用户直接调用基础检查类或内容生成类skill，完成单一任务。

**示例**：
```
用户：检查这个开发指南文档的质量
Agent：调用 checking-docs-guide-quality skill
输出：检查报告和改进建议
```

#### 4.5.2 组合使用模式
用户调用综合处理类skill，skill内部自动调用多个基础skill完成任务。

**示例**：
```
用户：根据d.ts文件生成开发指南
Agent：调用 generating-dev-guide skill
       ├─ 调用 checking-docs-guide-quality 检查规范
       ├─ 调用 generating-doc-section 生成内容
       └─ 调用 checking-version-consistency 检查一致性
输出：符合规范的完整开发指南
```

### 4.6 实施策略

#### 第一阶段：基础原子能力
1. 开发 `checking-docs-guide-quality` skill
2. 开发 `checking-sample-code-quality` skill
3. 开发 `checking-version-consistency` skill

#### 第二阶段：内容生成能力
1. 开发 `generating-doc-section` skill
2. 开发 `generating-sample-code` skill
3. 开发 `generating-faq-entry` skill

#### 第三阶段：综合处理能力
1. 开发 `generating-dev-guide` skill（组合基础skill）
2. 开发 `generating-systemapi-doc` skill（组合基础skill）
3. 开发 `maintaining-sample-code` skill（组合基础skill）

### 4.7 Skill 7: checking-docs-guide-quality

#### 用途
开发指南文档质量检查，采用"规则检查 + AI辅助 + 人工审核"的混合模式，实现对文档结构、内容完整性、内容质量、格式规范和合规性的全面检查。

#### 输入
- 待检查的Markdown文档路径
- 具体问题描述（可选，用于智能路由模式）

#### 输出
- 检查报告（JSON格式）
  - 规则检查结果（自动）
  - AI辅助检查结果（需人工确认）
  - 人工审核提示

#### 功能

##### 1. 问题分析引擎
- 识别问题类型（文档结构、内容完整性、内容质量、格式规范、合规性）
- 提取关键词
- 定位问题位置（行号、章节）

##### 2. 检查路由器
- 根据问题类型匹配相关规范
- 从统一规范库中提取规范要求

##### 3. 规则检查器（自动执行）
| 检查项 | 检查方式 |
|--------|----------|
| 标题层级 | 解析Markdown，检查标题等级 |
| 章节完整性 | 检查特定标题是否存在 |
| 标点符号 | 正则匹配 |
| 成对符号 | 正则匹配 |
| 段落长度 | 统计字数 |
| 敏感信息打码 | 正则匹配手机号、IP、域名 |

##### 4. AI辅助检查器（LLM执行）
| 检查项 | 检查方式 |
|--------|----------|
| 内容清晰性 | LLM分析文本，识别模糊、晦涩表述 |
| 术语规范性 | LLM检查术语使用是否一致 |
| 描述一致性 | LLM检查相似信息描述方式 |
| 场景覆盖完整性 | LLM判断场景是否完整 |
| 步骤描述清晰性 | LLM检查步骤描述是否清晰 |
| 开发建议充分性 | LLM判断关键步骤是否有建议 |
| SCQA写法 | LLM检查概述段落结构 |
| 语句通顺性 | LLM检查语句是否通顺 |

##### 5. 人工审核提示
- 原理描述正确性：提示与实际功能实现对比验证
- 无抄袭：提示检查与友商文档对比
- 与实际功能实现一致性：提示需要外部知识验证

#### 工作模式

##### 模式1：全面检查模式
对文档进行全面检查，逐项验证规范库中的所有检查项。

##### 模式2：智能路由模式（推荐）
根据用户输入的具体问题，智能匹配相关规范，精准检查。

**示例**：
```
用户输入："指南简介中'模拟器支持情况'目录和'约束与限制'目录的层级有问题"

处理流程：
1. 问题分析引擎识别：这是"文档结构-标题层级"问题
2. 规范路由器匹配：约束与限制应为###（三级标题）
3. 规则检查器执行：发现文档中使用了####
4. 生成结果：问题定位 + 规范依据 + 修改建议
```

api_reference_spec:

    - section: 返回值说明

    - section: 错误码

```typescript
/**
 * @brief 接口简要描述

 * @returns 返回值说明

 */
```
