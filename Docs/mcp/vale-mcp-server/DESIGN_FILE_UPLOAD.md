# Vale MCP Server 方案2：文件上传功能设计方案

## 背景

在 HTTP 远程模式下，服务器只能检查服务器本地的文件，无法直接访问客户端的磁盘。

用户场景：
- 客户端在本地编写文档
- 希望使用远程 MCP 服务器的 Vale 检查能力
- 但服务器无法访问客户端的本地文件

## 演进方案

### 方案 A：本地模式（已实现）

客户端使用本地 MCP Server，检查本地文档。

**优点**：
- 无需网络传输
- 配置简单

**缺点**：
- 每个客户端都需要配置 Vale + MCP Server
- 无法共享服务器上的规则配置

### 方案 B：文件上传功能（本次设计）

客户端上传文件到服务器，服务器检查后返回结果。

**优点**：
- 客户端无需安装 Vale
- 可以共享服务器的规则配置
- 统一管理检查结果

**缺点**：
- 需要网络传输文件
- 需要处理安全问题

## 方案 B 详细设计

### 功能列表

| 工具 | 功能 | 参数 |
|------|------|------|
| `upload_file` | 上传文件到服务器 | file_name, content (base64) |
| `check_docs` | 检查文档 | file_path（已有） |
| `delete_file` | 删除上传的文件 | file_name |
| `list_files` | 列出已上传的文件 | - |

### 安全性设计

#### 1. 文件大小限制
- **限制**：单个文件最大 10MB
- **原因**：防止超大文件占用磁盘和内存

#### 2. 文件类型限制
- **允许**：`.md`, `.txt`, `.json`, `.xml`, `.yaml`, `.yml`, `.csv`
- **原因**：只允许文档类文件，防止上传可执行文件

#### 3. 路径遍历防护
- **措施**：禁止文件名包含 `..`, `/`, `\`
- **验证**：保存后验证路径在允许目录内

#### 4. 文件覆盖保护
- **措施**：不允许覆盖已存在文件
- **原因**：防止意外覆盖重要文件

#### 5. 内容安全检查
- **措施**：检测可疑内容（如 PHP, JavaScript, eval 等）
- **原因**：防止上传恶意脚本

#### 6. 目录访问限制
- **措施**：通过 `VALE_ALLOWED_DIR` 限制可访问目录
- **原因**：确保服务器只能访问指定目录

#### 7. API 认证
- **措施**：HTTP 模式需要 API Key 认证
- **原因**：确保只有授权用户才能使用服务

### 稳定性设计

#### 1. 错误处理
- 所有文件操作都有错误处理
- 详细的错误信息返回给客户端

#### 2. 资源清理
- 提供 `delete_file` 工具手动删除
- 文件名限制防止目录遍历攻击

### 高并发设计

#### 1. 速率限制
- **措施**：每 IP 每分钟最多 60 次请求
- **原因**：防止恶意请求耗尽资源

#### 2. 无状态设计
- 不保存会话状态
- 可水平扩展

## 使用流程

```
1. 客户端：上传文件
   → upload_file(file_name, content)

2. 服务器：保存文件到允许目录
   → 返回成功消息

3. 客户端：检查文件
   → check_docs(file_name)

4. 服务器：执行 Vale 检查
   → 返回检查结果

5. 客户端：（可选）删除文件
   → delete_file(file_name)
```

## 配置示例

### 服务器端

```powershell
$env:VALE_ALLOWED_DIR="E:\ai\Jarvis\Docs"
$env:MCP_API_KEY="your-secure-key"
$env:MCP_SERVER_PORT="8080"
.\vale-mcp-server-http.exe
```

### 客户端（OpenCode）

```json
{
  "mcp": {
    "vale-mcp-server": {
      "type": "remote",
      "url": "http://192.168.1.100:8080/mcp",
      "headers": {
        "Authorization": "Bearer your-secure-key"
      }
    }
  }
}
```

### 使用示例

```
请上传并检查我的文档 README.md 的风格

（AI 会自动调用 upload_file 上传文件，然后用 check_docs 检查）
```
